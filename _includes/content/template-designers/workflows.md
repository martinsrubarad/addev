Workflows define the lifecycle of documents generated by ActiveDocs, encompassing drafting, approval, and finalization stages. Workflows can be configured globally, per subsite, or for specific Templates.

## What Is a Workflow?

A Workflow is a Design Component that governs how documents move through their lifecycle after generation. The three stages of a Workflow are:

- **Drafting** — Controls where and how draft documents are saved, including file format, naming, and storage location.
- **Approval** — Configures whether and how documents are routed for approval before finalization, including support for external approval services.
- **Finalization** — Defines how approved or completed documents are saved in their final form, including format conversion, delivery, and retention.

Workflows can be assigned at three levels:

- **Global** — Applies to all Templates across the entire ActiveDocs environment unless overridden.
- **Subsite** — Applies to all Templates within a specific subsite, overriding the global Workflow.
- **Template-specific** — Applies to an individual Template, overriding both global and subsite Workflows.

## Creating a Workflow

To create a Workflow:

1. In ActiveDocs WorkCenter Client, navigate to the **Workflows** node.
2. Right-click and select **New > Workflow**, or use the **New** button on the ribbon.
3. The **Workflow Properties** dialog opens with multiple configuration tabs.

### General Tab

| Option | Description |
| --- | --- |
| **Name** | The name of the Workflow as it appears in the Workflows tree and in assignment dialogs. |
| **Description** | An optional description of the Workflow's purpose and scope. |
| **Global** | When enabled, designates this Workflow as the global default. Only one Workflow can be global at a time. |
| **Enabled** | When enabled, the Workflow is active and available for assignment. Disabled Workflows are retained but not applied. |

### Categories Tab

Assign the Workflow to one or more Categories for organizational and filtering purposes.

### Approval Tab

The Approval tab configures whether and how documents are routed for approval.

| Option | Description |
| --- | --- |
| **No Approval** | Documents are not routed for approval. They proceed directly from drafting to finalization (if finalization is enabled). |
| **External Approval** | Documents are routed to an external approval service. Supported services include **SharePoint** approval workflows and **DocuSign** electronic signature workflows. |
| **ActiveDocs Approval** | Documents are routed for approval within the ActiveDocs system using built-in approval functionality. |

#### ActiveDocs Approval Options

When ActiveDocs Approval is selected, the following sub-options are available:

| Option | Description |
| --- | --- |
| **All Approvers** | All users with approval rights in the relevant scope can approve the document. |
| **Only these Approvers** | Only the specified users or groups can approve the document. Select approvers from the available list. |
| **Enable staged approval** | When enabled, approval proceeds in stages. Approvers in the first stage must approve before the document advances to the next stage. |
| **All must authorize** | When enabled, all designated approvers must approve the document. When disabled, a single approval is sufficient. |

#### Email and Notification Options

| Option | Description |
| --- | --- |
| **Email options** | Configure email notifications sent to approvers when a document is submitted for approval. |
| **Manual send** | When enabled, the approval notification email is not sent automatically. The document creator must manually trigger the notification. |
| **Approver can approve own** | When enabled, the user who created the document can also approve it. When disabled, a different user must approve. |
| **Creator can choose** | When enabled, the document creator can select the approver(s) at the time of submission. |
| **Default Approver** | Specifies a default approver who is pre-selected when the document is submitted for approval. |

### Draft Options Tab

The Draft Options tab controls where and how draft documents are saved.

#### Draft Save Location

| Option | Description |
| --- | --- |
| **Location Type** | Specifies the type of storage for draft documents. Available location types include ActiveDocs internal storage, file system paths, SharePoint document libraries, and other supported destinations. |
| **SharePoint destination** | When SharePoint is selected as the location type, click **Build URL** to construct the SharePoint library URL using the Build URL dialog. The dialog supports static URLs and dynamic URLs built from Active Field values. |

#### Save Options

| Option | Description |
| --- | --- |
| **Save Options for duplicates** | Configures behavior when a draft with the same name already exists at the save location. Options include overwriting the existing file, appending a numeric suffix, or prompting the user. |
| **Credentials** | Specifies the credentials used to authenticate when saving to the draft location. Options include default credentials, pre-set named credentials, or explicit username and password. |
| **Add ActiveDocs Columns** | When saving to SharePoint, adds ActiveDocs metadata columns to the SharePoint document library for tracking and filtering purposes. |

#### Draft Document Format and Navigation

| Option | Description |
| --- | --- |
| **Draft save format** | Specifies the file format for saved drafts (e.g., DOCX, PDF, or the Template's native format). |
| **Draft Document Wizard Finish URL** | Specifies a URL that the user is redirected to after saving a draft. This can be used to navigate the user to a custom page or back to the WorkCenter. |
| **Recreate Document Set mode** | When enabled, reopening a draft recreates the full Document Set rather than resuming from the saved state. |
| **Add new version** | When enabled, saving a draft creates a new version rather than overwriting the existing draft. |

#### Draft Status Indicator

The draft status indicator allows the document to display a visual marker indicating its draft status.

| Option | Description |
| --- | --- |
| **Property Name** | The document property name used to store the draft status value. |
| **Draft Text** | The text value stored in the property when the document is in draft status. |
| **Final Text** | The text value stored in the property when the document is finalized. |

#### Watermark Options

| Option | Description |
| --- | --- |
| **Watermark Text** | Text displayed as a watermark on draft documents (e.g., "DRAFT"). The watermark is removed upon finalization. |
| **Watermark Picture** | An image displayed as a watermark on draft documents. The watermark image is removed upon finalization. |

### Finalization Tab

The Finalization tab controls how documents are processed and stored when they reach their final state.

#### Finalization Settings

| Option | Description |
| --- | --- |
| **Document Finalization Enabled** | When enabled, the finalization stage is active for this Workflow. When disabled, documents remain in draft status after the Wizard is completed. |
| **Enable Document Set Finalization** | When enabled, all documents in a Document Set are finalized together as a unit. |
| **Automatically Finalize** | When enabled, documents are finalized automatically upon completion (or upon approval, if approval is configured). When disabled, finalization must be triggered manually. |
| **Document save location** | Specifies where finalized documents are stored. Configuration is similar to the draft save location, supporting ActiveDocs internal storage, file system paths, SharePoint, and other destinations. |
| **Save format** | Specifies the file format for finalized documents. |
| **Finish Page URL** | Specifies a URL that the user is redirected to after finalization is complete. |
| **Deliver Document copy** | When enabled, a copy of the finalized document is delivered via the configured delivery mechanism (e.g., email, print, or external system). |
| **Recreate Final using Answers** | When enabled, the finalized document can be regenerated from the saved answers, producing a fresh copy from the Template. |
| **Email notifications** | Configures email notifications sent when a document is finalized. Recipients and content are defined in the Email Templates tab. |

#### Document Options

Click **Document Options** to configure finalization document updates:

**Finalization Document Updates:**

| Option | Description |
| --- | --- |
| **Delete highlighting** | Removes all text highlighting from the finalized document. |
| **Delete hidden text** | Removes all hidden text from the finalized document. |
| **Delete comments** | Removes all comments from the finalized document. |
| **Accept revisions** | Accepts all tracked changes (revisions) in the finalized document, producing a clean final version. |

**Finalization Retention:**

| Option | Description |
| --- | --- |
| **Remove docs** | Removes draft documents from the draft save location after finalization. |
| **Remove file** | Removes the source file associated with the document after finalization. |
| **Remove previous versions** | Removes previous versions of the document, retaining only the final version. |
| **Reassign to Subsite** | Reassigns the finalized document to a different subsite. |
| **Reassign to User** | Reassigns ownership of the finalized document to a different user. |
| **Email notification** | Sends an email notification when retention actions are performed. |

### Retention Policy Tab

The Retention Policy tab configures how long finalized documents are retained before they are eligible for removal.

| Option | Description |
| --- | --- |
| **Do not apply** | No retention policy is applied. Documents are retained indefinitely unless manually removed. |
| **Retain forever** | Documents are explicitly marked for permanent retention. They are never automatically removed. |
| **Apply Retention Policy** | Applies a retention policy with configurable options for retention duration, review cycles, and disposal actions. Configure the specific retention period and disposal behavior as required by organizational policy. |

### Email Templates Tab

The Email Templates tab defines the email content used for notifications throughout the Workflow lifecycle. Seven email template types are available:

1. **Approval Request** — Sent to approvers when a document is submitted for approval.
2. **Approval Granted** — Sent to the document creator when the document is approved.
3. **Approval Declined** — Sent to the document creator when the document is declined.
4. **Finalization Complete** — Sent when a document is finalized.
5. **Draft Saved** — Sent when a draft document is saved.
6. **Retention Warning** — Sent before a retention action is executed.
7. **Retention Action** — Sent when a retention action is performed.

To customize an email template:

1. Select the email template type from the list.
2. Click **Edit** to open the email template editor.
3. Modify the subject line, body content, and recipient list as needed. Email templates support merge fields that insert document-specific values (such as document name, creator, and dates).
4. Click **OK** to save the customized template.

To restore an email template to its default content, select the template and click **Reset**.

### Subsites Tab

The Subsites tab associates the Workflow with specific subsites. When a Workflow is assigned to a subsite, it applies to all Templates within that subsite unless overridden by a Template-specific Workflow.

### Templates Tab

The Templates tab associates the Workflow with specific Templates.

To add a Template:

1. Click **Add** to open the Template browser.
2. Browse to and select the Template(s) to associate with this Workflow.
3. Click **OK** to add the selected Templates.

Templates listed on this tab use this Workflow regardless of any global or subsite Workflow assignments.
