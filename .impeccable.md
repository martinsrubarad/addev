# Impeccable Design Context — ActiveDocs Documentation

## Design Context

### Users
Enterprise IT teams, enterprise architects, independent consultants, ActiveDocs software users, and prospective clients evaluating the software. They access the documentation primarily at a desk during business hours, with mobile as a secondary means of access. They are looking for clear, authoritative technical information — not marketing. The job to be done is: find the answer, understand the concept, or complete the integration task as efficiently as possible.

### Brand Personality
**Three words:** Intelligent, discreet, precise.

**Tagline:** Document Intelligence.
**Brand line:** Invisible complexity. Visible confidence.

**Voice & tone:** Concise but not cold. Emotionally steady. Polished and subtle. The brand presents as a trusted, calm expert — never flashy or attention-seeking. Occasional dry wit is acceptable but the documentation tone skews utilitarian and efficient above all.

**Emotional goals for docs:** Confidence (the user trusts the information), calm (nothing feels overwhelming), efficiency (they find what they need fast).

### Aesthetic Direction

**Visual tone:** Refined minimal — restrained, precise, typography-forward. The aesthetic is flat, geometric, and precise. No visual excess. Typography and whitespace do the heavy lifting.

**Theme:** Light mode default (dark-on-light). This is a professional documentation site consumed at desks during working hours. Light-on-dark sections may be used sparingly for hero areas or feature callouts, but the reading experience is light.

**Colour palette:**

| Role | Name | Hex | Usage |
|------|------|-----|-------|
| Primary dark | Obsidian | `#19191C` | Backgrounds, headings, primary text |
| Text on light | Obsidian 70% | `#434345` | Body/paragraph text on white |
| Text on dark | Obsidian 50% | `#B0B0B0` | Paragraph text on dark backgrounds |
| Light border | Obsidian 20% | `#E8E8E8` | Borders, subtle backgrounds |
| Primary light | White | `#FFFFFF` | Backgrounds, text on dark |
| Secondary | Cerulean | `#23868E` | Accents, links, illustrations |
| Secondary soft | Cerulean 70% | `#6E9AA0` | Softer accent variation |
| Secondary tint | Cerulean 30% | `#B5D6D8` | Light tint for backgrounds, cards |
| Accent | Gold | `#A48266` | Accent highlights, CTAs, warm details |

**Colour rules:**
- Obsidian + White is the dominant pairing. Build on Obsidian.
- Cerulean and Gold are supporting — for accents, hover states, or illustration. They must never dominate.
- Tint neutrals toward Cerulean (hue ~185 in OKLCH) for subconscious brand cohesion.
- No pure black (`#000`) or pure white (`#fff`) in UI surfaces — use Obsidian and near-white tinted toward the brand hue.

**Typography:**

| Role | Font | Fallback |
|------|------|----------|
| Headings, hero statements, pull-out quotes | TWK Ghost Regular | Garamond, 'Times New Roman', serif |
| Body, navigation, buttons, UI copy | Helvetica Neue | Helvetica, Arial, sans-serif |

Note: TWK Ghost is a licensed font from WELTKERN. For the documentation site (Jekyll/GitHub Pages), the web font must be self-hosted or loaded via a font service. If TWK Ghost is not available as a web font, fall back to Garamond. Do not substitute with reflex-list serifs (Playfair, Cormorant, Lora, etc.).

**Anti-patterns (explicit brand don'ts):**
- No gradients, drop shadows, or decorative noise
- No flashy or attention-seeking elements
- No cyan-on-dark, purple-to-blue gradients, neon accents
- No glassmorphism, sparklines-as-decoration, or gradient text
- No side-stripe borders on cards
- No large rounded-corner icons above every heading

**Graphic elements:** Intersecting geometric shapes — overlapping rectangles and document-like forms. Subtle and structural, reinforcing the "discreet" personality. Used as section dividers or decorative backgrounds, never dominant.

**Photography (if used):** Paper and paper-craft motifs for marketing imagery. Warm, high-density shadows for people. Avoid flat, over-lit corporate stock.

### Design Principles

1. **Utility first.** The documentation exists to answer questions and unblock work. Every design decision serves findability and readability. Decoration that doesn't aid comprehension is removed.

2. **Quiet confidence.** The interface communicates authority through restraint — generous whitespace, precise typography, and a controlled palette. It never shouts.

3. **Invisible complexity.** The underlying product handles sophisticated document automation. The documentation mirrors this: complex information presented simply, with progressive disclosure where needed.

4. **Obsidian + air.** The visual system is built on the tension between Obsidian's weight and abundant white space. Cerulean and Gold appear only as punctuation — a link colour, a hover state, a call-to-action.

5. **Desktop-first, mobile-aware.** The primary context is a desk with a large screen. Layouts should use the available width intelligently (multi-column cards, wide content areas). Mobile receives a clean single-column adaptation, not a degraded experience.

### Technical Constraints

- **Platform:** Jekyll static site on GitHub Pages with Minimal Mistakes remote theme
- **CSS:** Single SCSS file (`assets/css/main.scss`) with theme imports and custom overrides
- **Fonts:** System stack currently; TWK Ghost would need to be self-hosted if available
- **No JS framework:** Plain HTML/Markdown with Liquid templating
- **Responsive breakpoints in use:** 900px (2-col), 600px (1-col)
